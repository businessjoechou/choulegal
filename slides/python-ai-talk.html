「點按鈕」的天花板

圖形介面，到哪裡會卡住？

  • 一次只能處理一筆——一萬筆呢？手動複製貼上一萬次，還是寫一個迴圈？

II
優勢
The leverage you gain
槓桿一 · Automation

把「做一次」變成「自動做無數次」

人做事是線性的：做十次花十倍時間。
程式做事是一次寫好、無限重播。

排程半夜自己跑、新資料進來自動更新——
你睡覺時它還在替你工作。

槓桿二 · Scale

從一萬列，到一千萬列

Excel 拉到幾十萬列就開始卡。
程式處理千萬列，是同一段邏輯、多等幾秒的事。

真實世界的問題，往往大到只能用程式碰。

槓桿三 · Reproducibility

程式是會自我記錄的流程

滑鼠點過的步驟，明年的你、隔壁的同事，都復原不了。

程式碼本身就是完整、可重跑的說明書。
科學要可重現，決策要可追溯——這是基本功。

槓桿四 · Integration

把 AI 接進真實系統

AI 的價值不在對話框裡，在它能串起其他東西：

你的資料庫、地圖 API、即時感測器、報表、通知——
用程式把它們接起來，AI 才從玩具變工具。

槓桿五 · Control

不被工具的預設值綁死

現成工具給你 80%。
剩下最關鍵的 20%——客製邏輯、特殊規則、你領域才懂的細節——
只有會程式的人改得動。

LIVE DEMO

現在，我們用 30 行
做一個小 AI

目標：把 1000 則使用者回饋，
自動分成正面 / 負面 / 中立，再畫成一張圖。

人工做要一整天。看程式要多久。

DEMO · 第一步：叫 AI 判斷一則
sentiment.py
import pandas as pd
from openai import OpenAI

client = OpenAI()
df = pd.read_csv("feedback.csv")   # 一欄 text，共 1000 則

def judge(text):
    r = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user",
            "content": f"判斷情緒，只回 正面/負面/中立：\n{text}"}])
    return r.choices[0].message.content.strip()

這段就是整個 AI 的核心——把一段文字丟給模型，拿回一個判斷。看懂這 10 行，你就懂「呼叫 AI」是怎麼回事了。

DEMO · 第二步：讓它跑完 1000 則並畫圖
sentiment.py（續）
# 一行，套用到全部 1000 則 —— 這就是「自動化」
df["情緒"] = df["text"].apply(judge)

# 統計各類有幾則
print(df["情緒"].value_counts())

# 一行畫成長條圖
df["情緒"].value_counts().plot.bar()

關鍵就是被框起來的那一行 .apply(judge)——「對每一則都做一次」。
人要做一千次的事，程式用一行交代完。

最強的學習加速器

AI 本身，就是你
24 小時的程式家教

  • 看不懂的錯誤訊息，貼給它問「這是什麼意思」

三個常見誤解

先把這些丟掉

誤解

「我數學不好，學不了。」 → 日常程式，九成是邏輯與耐心，不是數學。

誤解

「我不是資工系。」 → 最強的應用，常來自「懂領域 + 會一點程式」的人，那就是你。

誤解

「AI 以後會取代寫程式。」 → AI 讓寫程式更容易，於是會指揮它的人更值錢，不是更不值錢。
