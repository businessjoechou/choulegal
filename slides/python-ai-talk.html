程式不會因為 AI 消失。

「快去學寫程式，

今天想跟你們聊聊，
哪一半對，哪一半被誇大了。

先說一下我是誰

我不是工程師，

我只是學會了一件事 - 讓 Python 和 AI 幫我做事。

一個人，做出了三條法律科技產品線。今天分享的是這條路上的觀察，不是什麼結論 - 你們可以當場挑戰我。

I
挑戰
AI 改寫了什麼規則
你們心裡在想的那個問題

「我又不要當工程師，
學 Python 到底要幹嘛？」

答案很直接 - Python 在你的職涯裡不是主角，是外掛。

它不是讓你轉行當工程師。是讓你這個交管人，多一層別人拿不走的能力。

所以主題的答案

重點不是「把你變成工程師」。
是把你變成「會用程式的交管人」。

AI 接手了大部分的「動手寫」。
程式的重點，從寫得出來，移到了
看得懂、指揮得了、判斷得出對不對。

以前程式是你的手。
現在 AI 是手 -
程式變成腦。
誤會 · 一

「學會寫程式，
就不會被取代。」

AI 現在自己就會寫程式。光是「會語法、會打字」，已經不算什麼了 - 會寫，是基本款，不是護城河。

誤會 · 二

「會用 AI 就好，
幹嘛還懂程式。」

不懂的時候，你看不出它什麼時候在騙你。它常常一臉自信，給你一個其實不存在的答案 - 我自己每天都還要回去逐條核對。

誤會 · 三

「這是資工的事，
跟我科系無關。」

AI 不會只待在資工系。它進得了法律，也進得了交通、會計、醫療。它進的是每個領域，不分科系。

三個誤會收一下
門檻沒有不見，
它只是往上移了。

以前會一點皮毛就很吃香；那個空間會越來越小。
不是要嚇你們 - 是值得早一點知道。

II
工具
Python + AI 怎麼當你的外掛
AI 很強，但有三件事它做不了

看得懂、判斷得了、
抓得出錯

  • 拆問題 - 把一個模糊的需求，拆成機器做得了的小塊
  • 定標準 - 什麼叫「對」？那條線，要人來畫
  • 抓錯 - 它做完了，你看得出哪裡不對嗎？

懂一點 Python - 你就看得懂它在幹什麼。
懂你的領域 - 你就知道答案合不合理。
兩個加在一起，才叫判斷力。

講回你們 · 交通

純工程師不懂交通。
懂交通的人又不碰工具。
兩邊都沾一點的人，很少。

演算法算得出秒數，但居民抗議、施工限制、法規衝突，要人判斷。

物流 · 交通局 · 顧問

排班和路網可以自動化，但政策取捨和利害權衡，機器給不了答案。

你們的專業加上程式，是很稀有的組合。

classify.py
import pandas as pd
from anthropic import Anthropic

ai = Anthropic()
df = pd.read_csv("留言.csv")          # 1000 則使用者留言

def 分類(text):                       # ← AI 做的，就這一格
    r = ai.messages.create(model="claude-haiku-4-5", max_tokens=10,
        messages=[{"role":"user","content":f"只回 正面/負面/中立：\n{text}"}])
    return r.content[0].text.strip()

df["情緒"] = df["text"].apply(分類)      # 套用到 1000 則
df["情緒"].value_counts().plot.bar()   # 畫成圖

AI 只做框起來那一格 - 判斷一段文字的情緒。讀檔、套用一千次、畫圖，還有「結果到底對不對」，都是我這邊的事。

III
界線
AI 做不了的那條線
講個我自己的糗事

AI 引用了一條
根本不存在的法條

AI 的回答

「依《勞動基準法》第 84 條之 3 規定，雇主應......」

語氣超肯定、條號格式正確、讀起來完全沒問題。

事實

勞基法只到 §84-1、§84-2

§84-3 不存在。整條是 AI 自己編的。

我差點信了。直到我翻法條 - 才發現這條根本沒有。
如果我不懂法律，我不會去翻。如果我不會程式，我沒辦法寫一套東西逐條驗證。

所以我的系統，最重要的不是 AI 寫得多好

是每一條都回去對過

① 人定義問題

先決定要算什麼

哪些法條相關、計算邏輯怎麼走 - 這一步只有懂的人做得了。

② AI 生成初稿

讓 AI 跑第一版

它幫我整理條文、產出計算邏輯 - 但它的話，不能直接拿來用。

③ 人逐條核對

一條條回法規資料庫對

條號、修正日期、數字，每一筆我都回全國法規資料庫一條條對過。

人定義問題 → AI 跑初稿 → 人逐條核對 → 才有結論。
沒有什麼「系統算的所以一定對」這種事。

這是我每天最深的體會
AI 不是讓我可以「不懂」。
是讓我更需要「看得懂」。

它幫我做得更快 -
但能不能用，還是我說了算。

如果你也想開始

三個我自己會用的笨方法

  • 挑一份有資料的作業，跑一次 - 不用很難，一個 CSV 就夠與其把 Python 當考科，不如下次有資料的時候自己用 pandas 跑、畫張圖出來。
  • 找一個自己好奇的小題目 - 常騎的那條路到底多塞？拿真實資料做有興趣的題目，你才撐得過卡住的時候。交通類的開放資料很多。
  • 把 AI 當助教，但要追問它 - 問它「為什麼這樣寫」，別只複製貼上只抄答案其實沒學到什麼。追問才會。
